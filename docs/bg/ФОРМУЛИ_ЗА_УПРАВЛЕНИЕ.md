# Регистър на формулите за управление

Този документ е каноничният регистър на формулите по теория на управлението, използвани в кода.
За всяка формула се посочват: уравнение, символи/единици, източник и място в кода.

## Конвенции

- Външен интерфейс (телеметрия/CLI/рецепти):
  - PV: регулирана величина (температура) [°C]
  - SP: зададена стойност [°C]
  - OP: изход на регулатора [%]
- Вътрешни учебникови символи (формули/коментари):
  - y: изход на обекта [°C]
  - r: зададена стойност [°C]
  - u: управляващо въздействие към обекта [%]
- Съответствие:
  - y <-> PV, r <-> SP, u <-> OP
- e: грешка на регулиране = r - y [°C] (еквивалентно SP - PV)
- Ts, dt: период на дискретизация [s]
- K, tau, theta: усилване на FOPDT [°C/%], времеконстанта [s], закъснение [s]

## Смущение по изхода на регулатора

### Адитивно смущение в OP (само при control run)
- Уравнение:
  - `u_applied = clamp(u_ctrl + d_u(t), 0, 100)`
  - `d_u(t) = DIST_MAG_PCT` за активния интервал на смущението, иначе `0`
  - Активен интервал при режим STEP: `t >= DIST_START_S`
  - Активен интервал при режим PULSE: `DIST_START_S <= t < DIST_START_S + DIST_DURATION_S`
- Източник:
  - D. E. Seborg, T. F. Edgar, D. A. Mellichamp, F. J. Doyle III, *Process Dynamics and Control*, 3rd ed.
  - Стандартна адитивна форма на смущение по манипулираната величина при анализ на затворен контур
- Код:
  - `firmware/main.py::_disturbance_is_active`
  - `firmware/main.py` път за прилагане на OP в control loop

## PID семейство

### Паралелен PID с диференциална съставка по измерването
- Уравнение:
  - `u = clamp(P + I + D)`
  - `P = Kp*e`
  - `I[k] = I[k-1] + Ki*e*dt` (с политика за антинасищане)
  - `D = LPF(-Kd*d(PV)/dt)`
- Източник:
  - K. J. Astrom, T. Hagglund, *PID Controllers: Theory, Design, and Tuning*, 2nd ed.
- Код:
  - `firmware/control.py::PIDParallelPercent.update`

### Преобразуване SERIES/IDEAL
- Уравнение:
  - `Kc_eff = Kc*(1 + Td/Ti)`
  - `Ti_eff = Ti + Td`
  - `Td_eff = (Ti*Td)/(Ti + Td)`
- Източник:
  - K. J. Astrom, T. Hagglund, *PID Controllers*
- Код:
  - `firmware/control.py::_series_to_ideal_equivalent`

### PID с две степени на свобода (2DOF)
- Уравнение:
  - `e_p = beta*SP - PV`
  - `e_i = SP - PV`
  - `P = Kp*e_p`, `I += Ki*e_i*dt`, `D = -Kd*d(PV)/dt`
- Източник:
  - K. J. Astrom, T. Hagglund, *Advanced PID Control*
- Код:
  - `firmware/control.py::PID2DOFPercent.update`

## Релейна настройка

### Пределен коефициент от релеен тест
- Уравнение:
  - Релейният изход е фиксиран двупозиционен ON/OFF: `u_high=100%`, `u_low=0%`
  - `A = PV_pp/2`
  - `d_eff = (u_high - u_low)/2`
  - `Ku = 4*d_eff/(pi*A)`
  - `Pu = mean(период на трептения)`
- Източник:
  - Astrom, Hagglund (1984), DOI: `10.1016/0005-1098(84)90014-1`
- Код:
  - `firmware/identify.py::run_relay_tuning`

### Ziegler-Nichols PID (форма по пределен цикъл)
- Уравнение:
  - `Kp = 0.6*Ku`, `Ti = 0.5*Pu`, `Td = 0.125*Pu`
  - `Ki = Kp/Ti`, `Kd = Kp*Td`
- Източник:
  - Ziegler, Nichols (1942), DOI: `10.1115/1.2899060`
- Код:
  - `firmware/identify.py::run_relay_tuning`

### Tyreus-Luyben PID
- Уравнение:
  - `Kp = Ku/2.2`, `Ti = 2.2*Pu`, `Td = Pu/6.3`
  - `Ki = Kp/Ti`, `Kd = Kp*Td`
- Източник:
  - Tyreus, Luyben (1992), *ISA Transactions*
- Код:
  - `firmware/identify.py::run_relay_tuning`

### Избор P/PI/PID за релейни правила
- Уравнение:
  - `*_P`: `Ki=0`, `Kd=0`
  - `*_PI`: `Kd=0`
  - `*_PID`: без промяна на базовия PID комплект
- Източник:
  - Стандартна редукция на реда на регулатора от базово правило за настройка
- Код:
  - `firmware/identify.py::_rule_terms`
  - `firmware/identify.py::run_relay_tuning`

## Настройка по модел (FOPDT)

### Ziegler-Nichols 1 (reaction-curve, open-loop)
- Уравнение:
  - `ZN1_P:   Kc = tau/(K*theta)`
  - `ZN1_PI:  Kc = 0.9*tau/(K*theta),  Ti = 3.33*theta`
  - `ZN1_PID: Kc = 1.2*tau/(K*theta),  Ti = 2*theta,  Td = 0.5*theta`
  - Преход към паралелна форма: `Kp=Kc`, `Ki=Kc/Ti`, `Kd=Kc*Td`
- Източник:
  - J. G. Ziegler, N. B. Nichols (1942), DOI: `10.1115/1.2899060`
- Код:
  - `firmware/identify.py::_model_rule_set`
  - `firmware/identify.py::run_model_tuning`

### Cohen-Coon reaction-curve настройка от FOPDT
- Уравнение:
  - Дефиниция: `rho = theta/tau`
  - `CC_P`:
    - `Kc = (1/K)*(tau/theta)*(1 + rho/3)`
    - `Ki=0`, `Kd=0`
  - `CC_PI`:
    - `Kc = (1/K)*(tau/theta)*(0.9 + rho/12)`
    - `Ti = theta*(30 + 3*rho)/(9 + 20*rho)`
    - `Kd=0`
  - `CC_PID`:
    - `Kc = (1/K)*(tau/theta)*(4/3 + rho/4)`
    - `Ti = theta*(32 + 6*rho)/(13 + 8*rho)`
    - `Td = theta*(4/(11 + 2*rho))`
  - Преход към паралелна форма: `Kp=Kc`, `Ki=Kc/Ti`, `Kd=Kc*Td`
- Източник:
  - G. H. Cohen, G. A. Coon, *Theoretical Consideration of Retarded Control*, Transactions of the ASME, 1953.
  - D. E. Seborg, T. F. Edgar, D. A. Mellichamp, F. J. Doyle III, *Process Dynamics and Control*, 3rd ed. (таблица за reaction-curve настройка).
- Код:
  - `firmware/identify.py::_model_rule_set`
  - `firmware/identify.py::run_model_tuning`

## FOPDT идентификация

### Детектор за установено състояние (baseline и край на стъпката)
- Уравнение:
  - Условие за прозорец: `span(window) >= STEADY_WINDOW_S`
  - Условие за лента: `PV_p2p(window) <= 2*STEADY_BAND_C`
  - Установен baseline: условие за прозорец И условие за лента
  - Установено състояние след стъпка: базовите условия плюс защита за движение
    `|PV_avg_step - y0| >= 2*STEADY_BAND_C`
- Бележки:
  - Не се използват фиксирани min/max ограничения за време до установяване вътре в model идентификацията.
  - Callback-ите за stop/abort/overtemperature остават условия за прекратяване.
- Код:
  - `firmware/identify.py::_wait_for_steady`
  - `firmware/identify.py::run_test`

### Усилване на процеса
- Уравнение:
  - `K = (y_final - y_initial)/(u1 - u0)`
- Код:
  - `firmware/identify.py::run_test`

### FOPDT модел на отклик за RMSE
- Уравнение:
  - `y_hat(t) = y0` за `t < theta`
  - `y_hat(t) = y0 + K*(u1-u0)*(1 - exp(-(t-theta)/tau))` за `t >= theta`
- Код:
  - `firmware/identify.py::_simulate_fopdt_step`, `firmware/identify.py::run_test`

## ON/OFF регулатор

### Двупозиционно правило с хистерезис
- Уравнение:
  - ON ако `PV <= SP - hyst`
  - OFF ако `PV >= SP + hyst`
  - в противен случай се запазва предишното състояние
- Код:
  - `firmware/control.py::TwoPositionPercent.update`
