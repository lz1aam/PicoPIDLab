# Стил за отчети (Firmware + Lab)

Обхват
- Прилага се за runtime съобщенията в:
  - `firmware/*.py`
  - `runner/lab.py`

Класове съобщения
- `# PHASE:` започнат етап на изпълнение.
- `# INFO:` неутрална контекстна информация.
- `# WARNING:` нефатално отклонение.
- `# ERROR:` фатална грешка за текущото действие.
- `# RESULT:` изчислени/отчетни резултати.
- `# FINISH:` токен за приключване на процедура.
- `# READY:` токен за готовност/изчакване на команда.

Формат на блоковете
- Класифицираните блокове използват:
  - заглавен ред на класа (например `# RESULT:`),
  - разделител `# ========================================`,
  - детайлни редове с префикс `# `,
  - затварящ разделител.
- Не се използват алтернативни разделители.

Собственост на изхода
- Съобщенията от firmware са с префикс `# ...`.
- Съобщенията от host runner са с префикс `LAB: ...`.
- Host не генерира синтетични `# READY:` редове.

Правила срещу дублиране
- Не се повтаря еквивалентна информация в съседни редове.
- Контекстът за runtime е в един ред:
  - `runtime context: telemetry=<...> Ts=<...>s run_s=<...|none>` (поле `telemetry` е име от протокола)
- Host не добавя обобщение, ако firmware вече е изписал еквивалентен резултатен блок.

Редове за телеметрия
- Редовете за телеметрия не са класифицирани съобщения:
  - `PV:xx.x SP:xx.x OP:xx.x`
  - за MPC: `PV:xx.x SP:xx.x OP:xx.x YH:xx.x YP:xx.x`

Стабилност
- `# FINISH: done` е parser токен и се променя само при синхронна промяна на firmware и host parser.
- Семантиката на `# READY:` трябва да остане стабилна за коректна синхронизация на prompt.
